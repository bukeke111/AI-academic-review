# ARGP Demo v1.0.0 — 完整流程演示版

**AI Academic Review & Governance Platform（ARGP）** 静态前端 Demo，覆盖学生申报、秘书处治理、专家评审及公示异议等核心链路。

---

## 本版本亮点

### 秘书处退件
- 秘书处可对材料不合格项目执行退件，状态回滚至「草稿」
- 学生端首页待办、我的项目列表与详情页同步展示退件原因与修改指引
- 消息中心推送 AI 退件通知，支持一键跳转修改

### 评审结果申诉（学生端）
- 不予立项项目（`PROJ-2026-0078`）支持在项目详情侧栏提交申诉
- 申诉表单含理由、补充材料说明；提交后状态为「申诉受理中」
- 入口仅在项目详情页，不占用列表状态 Tab

### 异议与申诉（秘书处）
- 侧边栏新增 **「异议与申诉」** 工作台
- 秘书处桌面汇总待办：公示异议、结果申诉、专家回避
- 支持受理核查 / 受理复议 / 驳回 / 结案等操作
- 公示详情页可在线提交公示异议（`registerObjection`）

### 专家利益冲突与回避
- 利益声明中选择「同机构」等冲突情形后，流程改为 **提交回避申请**
- 隐藏重复的「拒绝邀请」面板，拒绝理由中不再出现「利益冲突」选项
- 回避申请与接受邀请分离，秘书处可在异议工作台查看

### 消息与状态样式
- 退件、申诉、异议相关消息统一接入消息中心（`actionFn` / `actionLabel`）
- 新增状态样式：`st-rejected`、`st-failed`、`st-appeal` 等

## 演示路径

| 角色 | 场景 | 操作 |
|------|------|------|
| 学生 | 退件待改 | 登录 → 首页待办 / 我的项目 → `PROJ-2026-0099` |
| 学生 | 结果申诉 | 我的项目 → 所有 → `PROJ-2026-0078` → 详情侧栏「提交申诉」 |
| 专家 | 利益冲突回避 | `review.html` → 专家登录 → 邀请页 → 声明冲突 → 提交回避申请 |
| 秘书处 | 异议与申诉 | `review.html` → 秘书处登录 → 「异议与申诉」或桌面汇总卡片 |
| 公众 | 公示异议 | 秘书处公示详情 → 提交异议表单 |

## 快速开始

```bash
# 克隆仓库
git clone https://github.com/bukeke111/AI-academic-review.git
cd AI-academic-review

# 启动本地静态服务（任选端口）
python3 -m http.server 8765
```

浏览器访问：

- 学生端：`http://localhost:8765/login.html`（选择「学生」角色）
- 评审端：`http://localhost:8765/review.html`（秘书处 / 专家 / 答辩等角色）

> 本 Demo 为纯前端 Mock，数据保存在浏览器内存与 `localStorage`，刷新后部分状态可能重置。
